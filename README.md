# Praveen Rajak — Portfolio

A cinematic 3D portfolio (Three.js + GSAP) with:

- **A hidden admin panel** — no link anywhere on the site. It lives only at a
  secret, unguessable URL you control (`/<ADMIN_ROUTE>`). Everything on the
  public site is editable from there: hero, about, CGPA, projects, experience,
  skills, contact, all links — and the LeetCode / Codeforces usernames.
- **Live DSA stats** — the DSA section pulls real numbers from your LeetCode and
  Codeforces public profiles (solved counts, ratings, difficulty split, and the
  Codeforces rating-over-time chart), cached ~30 min on the server.

## Stack

| | |
|---|---|
| Server | Node + Express |
| Database | MongoDB (Atlas free tier) via Mongoose |
| Auth | bcrypt password hash + signed httpOnly session cookie |
| Front-end | static `public/index.html` that fetches `/api/content` + `/api/dsa` |

```
public/index.html   the portfolio (renders from the API; static fallback baked in)
public/content.js   fetches content + live stats and patches the page before mount
admin/admin.html    the hidden editor (served ONLY at /<ADMIN_ROUTE>)
server.js           express app
src/                db, models, routes, services (leetcode/codeforces), seed
```

## 1. Configure

```bash
cp .env.example .env
```

Fill in `.env`:

- `MONGODB_URI` — from MongoDB Atlas (Create a free cluster → Connect → Drivers).
- `SESSION_SECRET` — `node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"`
- `ADMIN_USERNAME` / `ADMIN_PASSWORD` — your admin login (password ≥ 8 chars).
- `ADMIN_ROUTE` — your secret admin slug, e.g. `studio-7f3a91`. **Pick something
  no one can guess.** The panel is reachable only at `/<that slug>`; every other
  path returns a normal 404.

## 2. Seed (one time)

```bash
npm install
npm run seed
```

This creates the admin account (stored only as a bcrypt hash) and loads the
default content. Re-run anytime with `RESEED_CONTENT=1 npm run seed` to reset the
content back to defaults.

## 3. Run

```bash
npm start
```

- Portfolio: <http://localhost:3000/>
- Admin: `http://localhost:3000/<ADMIN_ROUTE>`  (type it yourself — no link exists)

## Deploy to Render

1. Push this repo to GitHub.
2. In Render: **New → Blueprint**, point it at the repo (uses `render.yaml`), or
   create a **Web Service** manually with build `npm install`, start `npm start`.
3. Set the env vars in the Render dashboard: `MONGODB_URI`, `ADMIN_ROUTE`,
   `ADMIN_USERNAME`, `ADMIN_PASSWORD`, and `NODE_ENV=production`
   (`SESSION_SECRET` is auto-generated by the blueprint).
4. After the first deploy, run the seed once — open the Render **Shell** for the
   service and run `npm run seed`.
5. (Optional, tidier) Once seeded you can delete `ADMIN_USERNAME` / `ADMIN_PASSWORD`
   from the env — they're only used by the seed script. Change your password later
   by re-running the seed with new values.

> In your MongoDB Atlas **Network Access**, allow connections from anywhere
> (`0.0.0.0/0`) or from Render's egress IPs, or the server can't reach the DB.

## Editing your content

Go to `/<ADMIN_ROUTE>`, sign in, edit any field, **Save changes**. The public
site reflects edits immediately on next load.

For DSA: set your LeetCode username and Codeforces handle on the **DSA** tab, then
**Save & refresh live stats** to pull the latest numbers (otherwise they refresh
automatically every ~30 minutes).

## Tests

```bash
npm run smoke   # boots an in-memory Mongo, exercises the API + drives a headless browser
```

## Security notes

- The admin route is security-through-an-unguessable-slug **plus** real
  credentials — both are required. Keep the slug private.
- Passwords are never stored in plaintext (bcrypt, cost 12).
- Login is rate-limited. Session cookies are httpOnly, SameSite=strict, and
  Secure in production.
- Admin-authored rich text (paragraphs / descriptions) is rendered as HTML by
  design so you can use `<b>`; only you can author it.
